## 3. V1.5 workstreams (scope: Phases 1–3)

### Phase 1 — Mobile Parity Closeout  *(first; pure UI over finished APIs)*
- **Review submission** — star-input bottom sheet on event detail + past-event prompt → `POST /api/protected/reviews`. Server enforces the attendance gate (403 NOT_ATTENDED / 409 ALREADY_REVIEWED); mobile just posts + handles those states. Haptics on star tap.
- **My Events screen** — Upcoming / Past / Cancelled tabs from `GET /api/protected/registrations?limit=50` (client-filtered by date, mirroring web `use-registrations`; `?status=cancelled` for the cancelled tab). New Profile sub-screen.
- **AI parity polish** — fully wire "Events Like This"; surface where summary/recs already exist.
- i18n en+am for all new strings (fold-forward). Ends in one APK.
- **Golden template:** `app/(tabs)/tickets.tsx` (FlatList + `useQuery` + `api.get` + async-storage cache) and `app/ticket/[id].tsx` (detail screen).

### Phase 3a — PostGIS `/nearby` route  *(small backend unit; unblocks Phase 2 GPS)*
- `GET /api/public/events/nearby?lat&lng&radius&limit` using `ST_DWithin`/`ST_Distance` on `events.coordinates`. Returns event cards + `distance`. (Sequenced before Phase 2 because Events Near Me depends on it.)

### Phase 2 — Mobile-Native Differentiators  *(the install-worthy moat)*
- **Biometric app-unlock** (`expo-local-authentication`) — opt-in Face ID / fingerprint gate in settings.
- **Events Near Me** (`expo-location`) — consumes Phase 3a; distance on cards; graceful denial.
- **Push loop polish** — verify token registration → `/devices` → notification **tap deep-links** to event/ticket.
- **Native share sheet** (`expo-sharing`) — upgrade RN `Share` (minor).
- New native modules → new preview APK. Guard the SDK-54 lockfile trap: root `npm dedupe` + root `npm ci --dry-run` before every EAS build.

### Phase 3b — Backend Automation (Edge Functions)  *(platform maturity)*
- **Waitlist auto-promotion** — promote-on-cancel trigger → ticket → notify.
- **AI Workflow Agents** as Supabase Edge Functions on cron/trigger: Reminder (24h/1h), Recap (24h post-event), Stale-Draft (14d), Trending (hourly featuring), Moderation auto-scan.
- Note: Recap/Reminder/Moderation agents only become *fully* meaningful once AI + comms are live (Phase 4). Build the plumbing now against stubs; they light up when keys flip.
